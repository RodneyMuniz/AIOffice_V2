# AIOffice Project

## Archive Status

AIOffice V2 is archived as of 2026-05-14.

R18 and R19 are closed as failed milestones. The project did not deliver the intended product vision and should not be treated as an active product, active milestone chain, or successful product runtime.

The repository remains available as reference material for future projects. The final archive marker is `PROJECT_ARCHIVE.md`.

This file is the founding charter and project identity for the clean reset repo.

## Charter
AIOffice is being rebuilt from a clean baseline as an admin-only, self-build-first product for governed software production.

This repo is a forward build, not a continuity archive. It preserves doctrine, naming discipline, authority patterns, and proven control rules from the reset bundle. It does not inherit legacy backlog history, milestone chains, or implied implementation claims from the donor repo.

## Project Identity
The project exists to prove that a narrow control harness can govern model-assisted software production without handing authority to the model.

The repo should stay small, explicit, and evidence-oriented until the first proof boundary is satisfied.

## Truth Surfaces
The authoritative truth substrates for the product are:
- Git history and Git-visible artifacts
- persisted state owned by the control model

Local drafts, model narration, and unreviewed workspace output are non-authoritative until reconciled against sanctioned truth surfaces.
