## Bounded R5 Planning Direction
The next bounded milestone should not be UI expansion, Standard runtime opening, or broad orchestrator productization. The next defensible move is to add the next foundation layer above R4 in three narrow focus areas:

1. Git-backed rollback and milestone baselines
2. Baton and resume model
3. CI/CD automation and repo enforcement

R5 should stay narrow and foundation-focused. It should define the next bounded layer needed after R4:
- trustworthy Git-backed milestone baseline capture
- explicit restore and rollback target model
- bounded rollback gate and authority model
- stronger baton semantics for re-entry and pause or resume continuity
- bounded resume path with explicit operator control
- stronger CI/CD automation
- stronger repo enforcement around proof, cleanliness, and governed change discipline

## R5 Non-Claims
Opening R5 should not imply:
- operator-visible UI work
- control-room productization
- chat or intake UX
- Standard or subproject runtime
- broad orchestrator autonomy
- unattended automatic resume
- broad recovery productization beyond bounded rollback and resume foundations
- cost dashboard productization
