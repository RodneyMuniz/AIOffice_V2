## 4. Open Operator Decisions Before Any Future Milestone Planning

These are decisions, not proposed tasks:

1. Approve or reject the updated constitutional thesis: board-driven, role-separated, memory-managed AIOffice.
2. Approve or adjust the 10 KPI domains and weights.
3. Decide whether GitHub Issues/Projects should be the first external board mirror.
4. Decide whether Linear should remain adapter/lab only until AIO board schema is proven.
5. Decide whether custom AIO board/control room remains target-state after schema proof.
6. Approve the RACI authority model, especially Auditor evidence authority and PM state ownership.
7. Approve the rule that Resolved is not Closed and User approval is required for closure.
8. Approve the knowledge classification model and cleanup approval path.
9. Approve the context/compaction control model before additional long Codex-driven cycles.
10. Approve release/environment strategy direction before promotion/closeout productization.
11. Approve security/safety/cost control categories before daemon/runner expansion.
