# R5 Git-Backed Recovery, Resume, and Repo Enforcement Foundations

## Milestone name
`R5 Git-Backed Recovery, Resume, and Repo Enforcement Foundations`

## Why this milestone exists now
R4 closed a bounded hardening milestone for internal control-kernel, workflow, and CI foundations. The first R4 closeout posture was not clean enough, and the corrective completion layer `R4-008` through `R4-011` was required before honest closure could be restated.

The next defensible step is still not UI expansion, not Standard or subproject runtime, and not broad orchestrator productization. The next bounded step is to implement the Git-backed recovery, resume, and repo-enforcement foundations that the repo did not prove at the end of R4.

## Objective
Strengthen the next bounded foundation layer after R4 by implementing Git-backed milestone baselines, bounded rollback and restore-gate foundations, stronger baton continuity and resume authority semantics, bounded resume re-entry, and stronger CI/CD proof-review plus repo-enforcement foundations without adding UI, without opening Standard runtime claims, without claiming automatic resume, and without claiming broader orchestration.

## Exit criteria
- repo truth records implemented bounded R5 foundations without widening the currently proved boundary beyond what code and proof justify
- Git-backed milestone baseline capture exists with explicit operator authority, clean-worktree requirements, and durable outputs
- bounded restore-gate results exist with explicit restore targets, authority checks, and fail-closed blocked reasons, but no restore execution claim
- strengthened baton continuity and resume authority are durable and explicit without claiming automatic resume
- bounded resume re-entry exists as an operator-controlled retry-entry path with invalid-state refusal behavior
- the next CI/CD proof layer is source-controlled and aligned to the actual bounded R5 proof scope
- repo-enforcement checks exist for clean worktrees, governed evidence presence, and bounded proof discipline
- the expected R5 proof and closeout structure is defined in advance, but milestone closure remains a later gated step

## In scope
- Git-backed milestone baseline capture model
- restore target and rollback gate authority model
- stronger baton continuity semantics for pause and resume re-entry
- bounded resume path expectations with explicit operator control
- stronger CI/CD automation direction beyond the current R4 bounded replay foundation
- stronger repo enforcement direction around cleanliness, governed evidence, and bounded proof discipline
- bounded proof and closeout structure for R5

## Explicitly out of scope
- any operator-visible or user-facing UI work
- unified workspace work
- chat or intake UX
- approvals queue UI
- cost dashboard UI
- settings or admin UI productization
- truthful visibility surface
- Standard or subproject runtime
- broad orchestrator autonomy
- unattended automatic resume
- rollback or broader recovery productization beyond the bounded foundation layer being defined here
- donor backlog import or donor milestone migration

## Dependencies and prerequisites
- `RST-009` through `RST-012` remain complete and externally accepted
- `R3-001` through `R3-008` remain complete in repo truth
- `R4-001` through `R4-011` remain complete in repo truth
- `governance/POST_R4_CLOSEOUT.md` and `governance/POST_R4_AUDIT_INDEX.md` remain the historical bounded R4 freeze surfaces
- Git and persisted state remain the authoritative truth substrates
- admin-only posture remains in force unless later repo truth explicitly proves more

## Key risks
- treating Git-backed baseline implementation as if rollback were already productized
- overstating baton continuity and bounded resume foundations as if automatic resume were already proved
- widening R5 into UI, Standard runtime, or broader orchestration before the underlying foundations are real
- letting CI/CD automation claims outrun the actual bounded proof scope
- allowing repo-enforcement implementation to turn into proof theater instead of verifiable discipline
